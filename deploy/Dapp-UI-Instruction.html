3b. Wallet Connect & Registration Flow

CRITICAL: Đây là màn hình đầu tiên user thấy. Phải hoạt động hoàn hảo trên MỌI thiết bị — từ iPhone SE (375px) đến 4K monitor. Tất cả form fields phải responsive, không bị tràn, không bị cắt.

3b.1 Flow tổng quan (3 bước)

1 Connect Wallet: RainbowKit modal → MetaMask / TrustWallet / WalletConnect
2 Registration Form: Ref. Link/Code (optional) + User ID (unique check) + Terms & Conditions
3 Sign & Enter: Sign nonce message → JWT issued → redirect to Dashboard
Logic: Nếu wallet đã đăng ký trước đó → skip bước 2, đi thẳng bước 3 (sign → dashboard). Bước 2 chỉ hiện cho user mới lần đầu kết nối.

3b.2 Registration Form — Components

FieldTypeRequiredValidationDetail
Ref. Link / Ref. Code Text input Optional Validate format: wallet address (0x...) hoặc code (alphanumeric). Check referrer exists on-chain. Auto-fill nếu user đến qua referral link (?ref=0x... hoặc ?ref=CODE123). Nếu đã có trong URL → hiển thị pre-filled + lock (không sửa được). Nếu trống → user có thể tự điền hoặc bỏ qua. One-time, immutable sau khi đăng ký.
User ID Text input REQUIRED 3–20 ký tự, a-z, 0-9, underscore. Realtime check trùng (debounce 500ms). ✅ Available / ❌ Taken. Tên hiển thị công khai. Unique across toàn hệ thống. Lưu PostgreSQL. Không thể thay đổi sau đăng ký (hiển thị warning).
Terms & Conditions Checkbox REQUIRED Phải checked trước khi submit. Button "Register" disabled nếu unchecked. Text: "I have read and agree to the Terms & Conditions and Privacy Policy". Link mở modal/new tab với nội dung đầy đủ.
Register Button Button (gold) — Disabled khi: User ID trống/trùng, T&C unchecked, hoặc đang loading. Click → call API POST /auth/register → lưu user → proceed to sign nonce → JWT → Dashboard.

3b.3 Responsive Rules (Tối quan trọng)

BreakpointLayoutForm WidthĐặc biệt
Mobile (< 640px) Full-screen overlay, no sidebar 100% width, padding 16px Input font-size ≥ 16px (tránh zoom trên iOS). Touch target ≥ 44px. Stack vertical.
Tablet (640–1024px) Centered card, max-width 480px 480px centered Logo + form trong 1 card. Background blur overlay.
Desktop (1024–1536px) Split layout: Left = branding / Right = form 420px form panel Left panel: Logo + tagline + animation. Right panel: form card.
Ultra-wide (> 1536px) Same as Desktop, max-width 1200px centered 420px form panel Thêm padding, font lớn hơn.
Responsive Principles:
• clamp() cho font sizes — không bị quá nhỏ/quá lớn
• min(100%, 480px) cho form container — tự co giãn
• Input fields: width: 100% — luôn full-width trong container
• Touch targets ≥ 44×44px trên mobile
• Không horizontal scroll — tuyệt đối không bao giờ
• Test trên: iPhone SE (375px), iPhone 14 (390px), iPad (768px), MacBook (1440px), 4K (2560px)

3b.4 API Endpoints (Registration)

EndpointMethodBodyResponse
/auth/check-userid GET ?userId=xxx { available: true/false }
/auth/check-referrer GET ?ref=0x... or ?ref=CODE { valid: true/false, referrerName: "..." }
/auth/register POST { walletAddress, userId, referrer?, termsAccepted } { success, nonce } → user signs nonce → /auth/verify → JWT
/auth/nonce GET ?wallet=0x... { nonce } (for returning users)
/auth/verify POST { walletAddress, signature } { jwt, user: { userId, wallet, referrer } }

5. User Flows — Critical Paths

5.1 New User Onboarding

1 Land on missionchain.io → see Connect Wallet CTA
2 Click Connect → RainbowKit modal → select MetaMask/TrustWallet
3 Sign message (nonce) → Backend verifies → JWT issued
4 Redirect to Dashboard → see empty portfolio with CTAs
5 Prompt: "Complete KYC to join SEED Round" or "Buy MICE to start mining"

5.2 SEED Purchase Flow

1 Navigate to /seed → KYC gate check
2 If not verified → redirect to Sumsub KYC flow
3 If verified + whitelisted → show 4 package cards
4 Select package → choose USDT or BNB payment
5 Approve USDT spending → SeedSale.purchase() → tx pending
6 Tx confirmed → show success + vesting schedule created

5.3 Mining Flow

1 Navigate to /mice → buy MICE license(s)
2 License activated → navigate to /mining
3 View Hindex score + daily emission share
4 Rewards accumulate daily (EmissionController.distributeDaily via keeper)
5 Click Claim → MiningPool.claimReward() → MIC to wallet

5.4 Staking Flow — Pure MIC Protocol

1 Navigate to Staking page (separate from NFT Manager)
2 Enter MIC amount + choose lock period (30/90/180/360 days)
3 See effective weight preview: amount × time-lock multiplier ONLY (×1.0 / ×1.25 / ×1.5 / ×2.0)
4 Approve MIC → Staking.stake() → position created
5 View position card with lock countdown + accrued rewards
6 Claim rewards anytime. Unstake max 10%/day after lock expires.
Key Architecture Changes:
• Staking is pure MIC: No NFT multipliers apply to staking. Rewards are based on amount × time-lock multiplier only.
• No staking caps: Users can stake unlimited MIC per address. (Previous per-NFT caps removed.)
• Locked MIC support: Vesting-locked MIC tokens (from SEED/Pre-Sale) can be staked with full time-lock multipliers (no reduction). Staking rewards are unlocked immediately and freely transferable. Minimum staking lock: 360 days.
• NFT multipliers moved: NFT multipliers (×1/×2.5/×5/×10) now apply ONLY to USDT reward pools (Community Builder Pool, Monthly NFT Pool, Lucky Draw), NOT staking.
