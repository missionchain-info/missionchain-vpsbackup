5. Login & Registration Flow — 3 Steps

Reference: Prototype-Connect-Wallet.html trong missionchain_app/ — đây là prototype đã được approved. Build chính xác theo flow và layout này.

5.1 Flow Overview

User visits missionchain.io
  │
  ├─ No wallet connected → Show LOGIN screen (full-page, no sidebar)
  │   │
  │   ├─ Step 1: CONNECT WALLET
  │   │   ├─ RainbowKit modal: MetaMask / TrustWallet / WalletConnect
  │   │   └─ On success → check if wallet registered
  │   │       ├─ YES (returning user) → Sign nonce → JWT → Dashboard
  │   │       └─ NO (new user) → Step 2
  │   │
  │   ├─ Step 2: REGISTRATION FORM
  │   │   ├─ Referral Code (optional) — auto-fill from URL ?ref=...
  │   │   ├─ User ID (required) — unique check, 3-20 chars, lowercase
  │   │   ├─ Terms & Conditions checkbox (required)
  │   │   └─ "Create Account & Sign" button
  │   │
  │   └─ Step 3: SUCCESS → Enter Dashboard
  │
  └─ Wallet connected + registered → Show DASHBOARD (with sidebar)

5.2 Registration Form — Field Specifications

FieldTypeRequiredValidationAPI
Referral Code / Link Text input Optional Wallet address (0x...) or alphanumeric code. Verify referrer exists. If from URL ?ref=: auto-fill + lock (disabled). GET /auth/check-referrer?ref=xxx
User ID Text input Required 3-20 chars, [a-z0-9_] only, force lowercase, realtime unique check (debounce 500ms). Cannot change after registration. GET /auth/check-userid?userId=xxx
Terms & Conditions Checkbox Required Must be checked. Register button disabled until checked. —

5.3 Auth API Endpoints

EndpointMethodRequestResponse
/auth/check-useridGET?userId=xxx{ available: boolean }
/auth/check-referrerGET?ref=0x... or ?ref=CODE{ valid: boolean, name?: string }
/auth/registerPOST{ wallet, userId, referrer?, termsAccepted }{ success, nonce }
/auth/nonceGET?wallet=0x...{ nonce }
/auth/verifyPOST{ wallet, signature }{ jwt, user }

5.4 Login Layout — Responsive

ViewportLayoutDetails
Desktop (>1024px) Split: Left branding | Right form Left: Logo + "Mission Chain" + tagline + stats (7B/100K/BSC). Right: step indicator + form. Max-width 960px.
Tablet (640-1024px) Centered card, max 480px Branding panel hidden. Mini brand header (logo + name) above form. Rounded card with backdrop blur.
Mobile (<640px) Full-screen, no border-radius Mini brand header top-left. Form fills screen. Padding 16px. Input 16px font (iOS). Touch target 48px.
